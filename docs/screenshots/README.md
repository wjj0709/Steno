# Steno 截图清单

本目录下的图片被仓库根目录的 [`README.md`](../../README.md) 直接引用。**按下表拍摄，并以表中的文件名保存到本目录，README 中的截图即会自动生效**，无需改动任何 Markdown。

在补齐之前，README 的「应用截图」区域会显示为图片占位（alt 文本），属正常现象。

## 通用规范

- **格式**：PNG。文件名必须与下表完全一致（小写、连字符）。
- **主题**：浅色 / 深色任选其一，但**全程保持一致**（建议浅色，与应用图标观感统一）。
- **内容脱敏**：不要出现真实笔记内容、本机用户名、真实文件路径。统一用示例数据（例如一条名为「Rust 知识点」的笔记、几条示例待办）。
- **清晰度**：主窗口类截图建议 ≥ 1440×900，或在 HiDPI / 2× 下截取后等比缩放；浮窗类按窗口实际尺寸即可。
- **窗口边缘**：尽量截取干净的窗口（含圆角），避免带入其他应用或杂乱桌面背景。
- **体积**：导出后可用 TinyPNG 等压缩，单图建议 < 500 KB。

启动应用以便截图：

```bash
pnpm tauri:dev
```

## 小贴士

- 想要更统一的观感，可把主窗口固定为同一尺寸（如 1280×800）后再逐个视图截图。
- 浮窗类（速记浮窗、今日待办浮窗）建议连同投影一起截，更能体现「悬浮在其他应用之上」。
- 截图完成后，回到根目录 `README.md` 预览，确认每张图都正常显示且宽度协调。
